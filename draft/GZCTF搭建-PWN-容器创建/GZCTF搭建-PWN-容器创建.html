GZCTF搭建 & PWN 容器创建

无标签
发布日期:   2025-02-11
文章字数:   593
阅读时长:   2 分

前置环境配置

安装 docker：

sudo apt update
sudo apt install docker.io docker-compose

换源：

vim /etc/docker/daemon.json

写入内容：

{
	 "registry-mirrors": [ "https://a0f4d5484dea442caf5d1193f09abdd4.mirror.swr.myhuaweicloud.com" ]
}

GZ::CTF搭建

  • 在根目录创建 GZCTF 文件夹，下载 GZCTF 并且存到根目录的 GZCTF 文件夹

  • 在 GZCTF 文件夹下创建 appsettings.json

    <Your POSTGRES_PASSWORD>：数据库密码

    <Your XOR_KEY>：任意字符串

    <Your PUBLIC_ENTRY>：服务器外部访问 ip

    CaptchaConfig：用于设置验证码，CloudflareTurnstile 网站中 add widget 后得到 SITE_KEY 、SECRET_KEY（None就不用填）

    {
  "AllowedHosts": "*",
  "ConnectionStrings": {
    "Database": "Host=db:5432;Database=gzctf;Username=postgres;Password=<Your POSTGRES_PASSWORD>"
  },
  "EmailConfig": {
    "SendMailAddress": "a@a.com",
    "UserName": "",
    "Password": "",
    "Smtp": {
      "Host": "localhost",
      "Port": 587
    }
  },
  "XorKey": "<Your XOR_KEY>",
  "ContainerProvider": {
    "Type": "Docker", // or "Kubernetes"
    "PortMappingType": "Default", // or "PlatformProxy"
    "EnableTrafficCapture": false,
    "PublicEntry": "<Your PUBLIC_ENTRY>", // or "xxx.xxx.xxx.xxx"
    // optional
    "DockerConfig": {
      "SwarmMode": false,
      "Uri": "unix:///var/run/docker.sock"
    }
  },
  "RequestLogging": false,
  "DisableRateLimit": true,
  "RegistryConfig": {
    "UserName": "",
    "Password": "",
    "ServerAddress": ""
  },
  "CaptchaConfig": {
    "Provider": "None", // or "CloudflareTurnstile" or "GoogleRecaptcha"
    "SiteKey": "<Your SITE_KEY>",
    "SecretKey": "<Your SECRET_KEY>",
    // optional
    "GoogleRecaptcha": {
      "VerifyAPIAddress": "https://www.recaptcha.net/recaptcha/api/siteverify",
      "RecaptchaThreshold": "0.5"
    }
  },
  "ForwardedOptions": {
    "ForwardedHeaders": 5,
    "ForwardLimit": 1,
    "TrustedNetworks": ["192.168.12.0/8"]
  }
}

  • 在 GZCTF 文件夹下创建 docker-compose.yml

    <Your POSTGRES_PASSWORD>：和 appsettings.json 中一样的数据库密码

    <Your GZCTF_ADMIN_PASSWORD>：初始管理员用户密码

    version: "3.0"
services:
  gzctf:
    image: registry.cn-shanghai.aliyuncs.com/gztime/gzctf:develop
    restart: always
    environment:
      - "GZCTF_ADMIN_PASSWORD=<Your GZCTF_ADMIN_PASSWORD>"
      # choose your backend language `en_US` / `zh_CN` / `ja_JP`
      - "LC_ALL=zh_CN.UTF-8"
    ports:
      - "80:8080"
    volumes:
      - "./data/files:/app/files"
      - "./appsettings.json:/app/appsettings.json:ro"
      # - "./kube-config.yaml:/app/kube-config.yaml:ro" # this is required for k8s deployment
      - "/var/run/docker.sock:/var/run/docker.sock" # this is required for docker deployment
    depends_on:
      - db

  db:
    image: postgres:alpine
    restart: always
    environment:
      - "POSTGRES_PASSWORD=<Your POSTGRES_PASSWORD>"
    volumes:
      - "./data/db:/var/lib/postgresql/data"

  • 启动

    docker-compose up -d

    关闭

    docker-compose down

  • 访问服务器 ip，可以看到 GZ::CTF 平台

创建管理员用户

进入之后可能是没有管理员用户的，就需要先注册一个账户再设置为管理员用户

设置管理员：username 替换为用户名

docker compose exec db psql -U postgres
\c gzctf
UPDATE "AspNetUsers" SET "Role"=3 WHERE "UserName"='username';

pwn容器创建

可以直接使用 docker 模板 换个 src 下的可执行文件（文件名改成 attachment）即可使用，将 docker 文件放到服务器中，在 docker 文件夹中执行 docker build t docker_name .（docker_name自定义）生成镜像，在创建 pwn 题的时候容器镜像填自定义的 docker_name，服务端口填 9999 （模板默认 expose 端口 9999）即可创建测试容器
